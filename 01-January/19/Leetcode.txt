1292. Maximum Side Length of a Square with Sum Less than or Equal to Threshold
Given a m x n matrix mat and an integer threshold, return the maximum side-length of a square with a sum less than or equal to threshold or return 0 if there is no such square.
________________________________________________________

Code :
class Solution {
    public int maxSideLength(int[][] mat, int threshold) {
        int rows = mat.length, cols = mat[0].length;

        for (int i = 0 ; i < rows ; i++) {
            for (int j = 1 ; j < cols ; j++) {
                mat[i][j] += mat[i][j-1];
            }
        }
        for (int i = 1 ; i < rows ; i++) {
            for (int j = 0 ; j < cols ; j++) {
                mat[i][j] += mat[i-1][j];
            }
        }
        
        int maxDiagLen = 0;
        for (int i = 0 ; i < rows ; i++) {
            for (int j = 0 ; j < cols ; j++) {
                for (int diagLen = maxDiagLen + 1 ; i + 1 - diagLen >= 0 && j + 1 - diagLen >= 0 ; diagLen++) {
                    int iPrev = i - diagLen, jPrev = j - diagLen;

                    int topLeft = iPrev >= 0 && jPrev >= 0 ? mat[iPrev][jPrev] : 0;
                    int left = jPrev >= 0 ? mat[i][jPrev] : 0;
                    int top = iPrev >= 0 ? mat[iPrev][j] : 0;

                    int sum = mat[i][j] + topLeft - top - left;
                    if (sum <= threshold) {
                        maxDiagLen = diagLen;
                    } 
                    else {
                        break;
                    }
                }
            }
        }
        
        return maxDiagLen;
    }
}
________________________________________________________

Intuition :
We need to find the largest square submatrix whose sum is less than or equal to a given threshold. Calculating the sum of every possible square repeatedly would be inefficient. A standard optimization is to use a 2D prefix sum matrix, which allows us to compute the sum of any submatrix in constant time. With this, we can efficiently try to expand squares and check whether their sums stay within the threshold, updating the maximum valid side length found.
________________________________________________________

Approach :
Convert the given matrix into a 2D prefix sum matrix:
First accumulate row-wise sums.
Then accumulate column-wise sums.

Use the prefix sum matrix to compute the sum of any square in O(1).
Iterate over each cell as the potential bottom-right corner of a square.
Try increasing square side lengths starting from the current maximum found so far.

For each candidate square, compute its sum using the prefix sum formula.
If the sum is within the threshold, update the maximum side length.
Stop expanding the square when the sum exceeds the threshold.

Return the maximum side length found.
________________________________________________________

Complexity :
Time complexity: O(m⋅n⋅min(m,n))
Space complexity: O(1)
